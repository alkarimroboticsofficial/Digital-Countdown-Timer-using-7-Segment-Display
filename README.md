# 🌱 ESP32 HT16K33 Adjustable Stopwatch & Timer

An ESP32-based **Adjustable Stopwatch and Countdown Timer** configured with an I2C HT16K33 7-Segment display, physical tactile control buttons, and a Piezo alert buzzer module.  
The system processes interactive state modifications allowing on-the-fly incrementing, decrementing, resetting, and toggling over pins 21 and 20.

---

## 📌 Project Overview

This project provides an advanced interactive solution for **industrial processing stops**, **laboratory testing clocks**, and **productivity workstations** by combining physical input processing logic with an ESP32 microcontroller.  
Users control state updates directly through an isolated 4-button external matrix network.

- Pressing **Add / Subtract** adjusts the target timing value up or down by 1 minute
- Pressing **Start / Pause** toggles the active countdown loop state
- Pressing **Reset** completely halts execution routines and wipes the duration variables back to zero

The system displays:

- Live minutes and seconds remaining
- Active run-time visual sync pulses via the blinking display colon
- Dedicated hardware buzzer audio tone bursts upon timeout limits ($00:00$)

on the high-contrast HT16K33 alphanumeric display backdrop.

# ⚙️ Working Principle

1. The starting runtime memory registers clear to an inactive tracking loop structure inside the setup stack.
2. Software-based debouncing filters continuous analog key noise from the inputs to prevent accidental multiple triggers.
3. Users press control triggers to append parameters, shift modes, or completely zero current counts.
4. Calculations split numerical variables into structured minute/second values and push them to the HT16K33 chip.
5. If the time counter parameters drop to zero ($00:00$) while in active processing mode:
   - System loop forces running states back to zero
   - Output channel Pin 42 cycles voltage frequencies to trigger the alarm
   - The Piezo buzzer generates an absolute timeout warning block
6. Paused or unconfigured execution cycles keep current metrics fixed until the next manual instruction override.

# 🔌 Pin Connections

Inputs use internal pull-up resistor structures, meaning one leg of the button wires to the pin and the other wires directly to GND.

## HT16K33 7-Segment Module

| HT16K33 Pin | ESP32 Pin |
| ----------- | ---------- |
| VCC         | 3V3 / 5V   |
| GND         | GND        |
| SDA         | GPIO21     |
| SCL         | GPIO20     |

| Reset Counter State    | GPIO46          |

# 🧠 ESP32 Code Logic

The ESP32 performs the following tasks:

- Scans hardware digital rails for logic state transitions generated by button presses
- Evaluates real-time elapsed durations utilizing non-blocking system millisecond functions
- Translates combined counting metrics into standalone numerical segments for minute/second grids
- Passes I2C communication arrays down onto internal HT16K33 registers via pins 21 and 20
- Handles localized short-pulsed frequency alarms across pin GPIO42 during time matches

---

# 📊 Time Matrix Calculation Formula

The separation of the cumulative runtime database variable into individual high and low display integers follows standard modulo arithmetic:

```math
Display\ Integer \(= \left( \lfloor \frac{Seconds_{total}}{60} \rfloor \times 100 \right) +\)(Seconds_{total} \(\bmod 60\))
```

# 🚀 Installation Steps

1. Build your component array layouts precisely utilizing the custom hardware pin mapping charts.
2. Install required Arduino IDE compiler frame attachments:
   - Adafruit_GFX.h
   - Adafruit_LEDBackpack.h
3. Choose your designated target ESP32 developer profile within the board settings panel.
4. Send the updated system executable code directly down onto your connected target hardware board.
5. Tap your control buttons to configure custom time boundaries and launch your timer.
